# 객체지향에서 클래스, 추상클래스, 인터페이스
* 누구에게나 상속 가능한 대상이다

## 클래스
* 상속을 받게되면 부모 클래스의 정의된 모든 method(private 제외)를 코드 추가 없이 그대로 사용 가능
* 부모 클래스의 정의된 method중 일부를 내 맘대로 다시 작성하여 나만의 코드로 사용을 목적

## 인터페이스
* 구체적인 실행 코드가 없는 클래스
* 모든 method는 이름만 존재하고 아무런 구현 코드가 없다
* 인터페이스를 implements(상속) 받은 클래스는 반드시 인터페이스에 정의된 method를 모두 구현해야할 의무가 있다
* 협업과정 업그레이드 과정에서 표준을 만들고 표준을 따르도록 하기위한 prototype을 정의하는 것과 동일
* 인터페이스를 상속받아서 사용하는 클래스들은 프로젝트에서 매우 효과적으로 여러 클래스를 보완하면서  
코드를 작동 할 수 있다

* 인터페이스를 상속받을 클래스들의 일부method가 변함이 없는 공통코드가 존재할때  
그러한 코드마저 다시 복붙 등으로 작성해야 하는 불편이 존재
* 인터페이스를 상속받아 구현된 method의 공통된 코드가 있으면 이런 코드는 재활용 하고픔,  
다만 인터페이스는 이것이 불가능

## 추상클래스(abstract class)
* 보편적인 클래스의 성질과 인터페이스의 성질을 모두 갖는 클래스
* 상속받은 클래스들이 공통으로 사용할수 있는 method는 미리 작성을 해두고  
  인터페이스처럼 일부는 상속받은 클래스가 다시 작성하도록 강제하는 규정(성질)을 갖는 클래스
